# ✅ 실제 구현된 CRUD 기능 목록

현재 **백엔드(Server)와 프론트엔드(UI)가 모두 연동되어 작동하는** 진짜 기능 목록입니다.

---

## 🔒 1. 회원 (Auth)
*   **회원가입 (`Create`)**: 사용자가 아이디, 비밀번호, 이름, 이메일, **역할(학생/강사)**을 입력하여 계정을 생성합니다. (비밀번호 확인 및 중복 검사 포함)
*   **로그인 (`Read`)**: 아이디와 비밀번호를 검증하여 세션을 발급받고 메인 화면으로 진입합니다.
*   **내 정보 조회 (`Read`)**: 마이페이지에서 나의 가입 정보와 역할(Role)을 확인합니다.

---

## 👨‍🏫 2. 강사 (Instructor) - 강좌 관리
*   **강좌 개설 신청 (`Create`)**: 강좌 제목, 설명, 난이도, 가격, 썸네일을 입력하여 생성을 요청합니다. (초기 상태는 **승인 대기**입니다.)
*   **내 강좌 목록 조회 (`Read`)**: 내가 개설한 강좌들만 모아서 상태(승인됨/대기중/반려됨)와 함께 확인합니다.
*   **강좌 상세 내용 조회 (`Read`)**: 특정 강좌를 클릭하여 상세 커리큘럼과 정보를 확인합니다.
*   **강좌 정보 수정 (`Update`)**: 강좌 제목이나 설명, 가격 등의 정보를 수정합니다. (본인 강좌만 수정 가능)
*   **강좌 삭제(버튼없음) (`Delete`)**: 실수로 만든 강좌나 필요 없는 강좌를 삭제합니다. (본인 강좌만 삭제 가능)

---

## 👮 3. 관리자 (Admin) - 승인 시스템
*   **승인 대기 목록 조회 (`Read`)**: 강사들이 신청한 강좌 중 아직 검토하지 않은(**Pending**) 강좌 리스트를 확인합니다.
*   **강좌 승인 (`Update`)**: 강좌 내용을 검토하고 "승인" 버튼을 눌러 **공개 상태(Approved)**로 변경합니다.
*   **강좌 반려 (`Update`)**: 강좌 내용이 부적절할 경우 사유를 적어 "반려" 처리합니다. (강사는 사유를 확인할 수 있습니다.)

---

## 👨‍🎓 4. 학생 (Student) - 수강
*   **수강 신청 (`Create`)**: 마음에 드는 강좌 상세 페이지에서 "수강 신청" 버튼을 눌러 내 강의실에 등록합니다.
*   **내 수강 목록 조회 (`Read`)**: 마이페이지에서 내가 현재 수강 중인 강좌 리스트를 확인합니다.

---

## 🧪 5. 테스트 기능 (Test Logic)
> *기능 로직은 완벽하나, UI가 개발자 테스트용 페이지(`/test/...`)에 있는 기능들입니다.*

*   **코딩 테스트 (`Create/Read`)**: 문제를 조회하고, 코드를 작성해서 실행(`Run`)하거나 제출(`Submit`)하면 채점 결과를 받습니다.
*   **커뮤니티 게시판 (`CRUD`)**: 게시글을 작성, 목록 조회, 상세 조회, 내용 수정, 삭제할 수 있으며 댓글도 마찬가지로 전체 기능을 지원합니다.
*   **Q&A 질문답변 (`CRUD`)**: 강의에 대한 질문을 올리고, 강사가 답변을 달거나 수정/삭제하는 기능입니다.
