# ✅ 전체 기준 (삭제 + 수강취소 포함)


* **수강취소** → 학습자 관점의 제어 기능
* **삭제(비활성화)** → 운영자 관점의 제어 기능

---

# 🟩 수강생 기능 우선순위 (수강취소 ⭕)

## 🔥 1순위 — 반드시 구현

👉 **코딩 학습이 실제로 한 사이클 도는 데 필요한 기능**

* 회원가입 / 로그인 ⭐
* 강좌 목록 / 상세 조회
* 수강 신청
* 수강 취소
* 강의 시청
* 출석 처리
* 학습 대시보드
* 코딩 문제 풀이
* 코드 제출 & 채점 결과
* 레벨업 시험 응시
* 시험 결과 확인

### 📌 수강취소 정책 (백엔드 기준)

* 강의 시작 전: 즉시 취소 가능
* 학습 진행 중: 취소 가능 (진도 기록 유지)
* 수강 완료 후: 취소 불가
* 수강 취소 시 출석 및 학습 이력은 보존된다.

---

## 🟡 2순위 — 부분 구현

👉 **LMS 완성도 보강**

* 출석 기록 조회
* 학습 이력 조회
* 시험 이력 조회
* Q/A 질문 등록 / 조회
* 알림 조회

---

## ⚪ 3순위 — 설계만

👉 **서비스 고도화 영역**

* 결제 / 환불 자동 연계
* 설정(프로필, 비밀번호)
* 수강 완료 / 예정 강좌 분리

---

# 🟨 강사 기능 우선순위 (수강취소 ❌ / 삭제 ❌)

## 🔥 1순위 — 최소 구현

👉 **강사는 ‘콘텐츠 제공자 + 요청자’**

* 강사 승인 상태 확인
* 강사 대시보드 조회
* 강좌 개설 요청
* 강의 업로드 요청

📌 강사는

* 수강 취소 ❌
* 강의 삭제 ❌

(운영 정책 일관성 유지)

---

## 🟡 2순위 — 부분 구현

* 과제 등록
* 과제 제출 현황 조회
* Q/A 답변 등록

---

## ⚪ 3순위 — 설계만

* 수익 / 정산
* 계좌 관리
* 강사 알림 상세

---

# 🟥 관리자 기능 우선순위 (삭제 ⭕)

## 🔥 1순위 — 반드시 구현

👉 **서비스 운영 통제 핵심**

* 관리자 로그인
* 회원 목록 조회
* 강사 승인 / 반려
* 강좌 승인 / 반려
* **강좌 비활성화(삭제 처리)** ⭐
* **강의 비활성화(삭제 처리)** ⭐
* **수강 취소 관리(관리자 강제 취소)** ⭐

### 📌 관리자 삭제/취소 원칙

* ❌ 물리 삭제
* ⭕ 상태 변경 기반 제어

---

## 🟡 2순위 — 부분 구현

* 강의 목록 조회
* 커뮤니티 게시글 숨김 처리
* 시험 관리 조회
* 로그 조회

---

## ⚪ 3순위 — 설계만

* 매출 통계 상세
* 알림 템플릿 관리
* 제재 이력 상세 관리

---

# 🧠 “삭제 + 수강취소” 정리 표 (최종)

| 기능    | 수강생 | 강사 | 관리자 | 방식    |
| ----- | --- | -- | --- | ----- |
| 수강 취소 | ⭕   | ❌  | ⭕   | 상태 변경 |
| 강의 삭제 | ❌   | ❌  | ⭕   | 비활성화  |
| 강좌 삭제 | ❌   | ❌  | ⭕   | 종료 처리 |

---

## 🎯 이 정리가 좋은 이유

* **학습 흐름 보존**
* 로그/이력 유지
* DB 구조 단순
* “운영 정책 이해도 높다”는 평가 받기 쉬움
